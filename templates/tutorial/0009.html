The Bigger They Are...

(λ)

The bigger function works in the same manner as smaller, except that it "kills" the lower limit.

Go ahead and type this in the interactions panel:

(define (bigger)
  (set! lower (min upper (add1 (guess))))
  (guess))

Now type lower in the interaction panel. It should return the lower limit that we initially set: 1

Now type: (bigger), and again type: lower

So, the set! expression in bigger:

  • increments the value returned by guess: (add1 (guess))
  • uses the min function to return the smallest of upper and (add1 (guess))
  • assigns the result of min to lower, setting a new lower limit

The bigger function administers the coup de grâce by calling the (guess) function.

bigger and smaller
are functions that contain functions. bigger and smaller are the enclosing functions, and the functions defined inside of them are called nested functions.
